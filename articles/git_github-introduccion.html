Git GitHub Introduccion

Como introduccion a git y github, tenemos que entender su uso y como nos puede ayudar a mejorar y facilitar nuestro trabajo como programadores y desarrolladores. Estas herrmaientas nos permiten tener un mayor control sobre nuestro codigo y sus versiones.

git nos tener un repositorio local, un repositorio es un espacio donde se almacena y se organiza la informacion del usuario. y con github lo que tenemos es un repositorio en la nube en la cual podemos tener distintas ramas o branchs y compartir git con otros desarrolladores.

Branch o rama en espa√±ol se pueden entender como las distintas versiones que podemos tener de un proyecto en la cual tenemos un branch principal que se crea al crear el repositorio y que generalmente tienen el nombre de main o master.

Las branch nos permiten por ejemplo tener un brnach por un lado en el que podemos trabajar solo una parte de la aplicacion web como por ejemplo la cabecera, el footer u otro elemento, y si resulta bien unirlo al branch principal o main haciendo uso de merge.

Dentro de git tenemos distintos modelos de trabajo que es la forma en la que usamos los branchs y como trabajamos con git entre ellos tenemos Trunk-based y gitflow.
